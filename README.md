# Ubuntu Terminal Trainer

Интерактивный симулятор консоли Ubuntu для изучения командной строки Linux прямо в браузере.
Бэкенд не нужен — работает как статический сайт (GitHub Pages, Vercel, Netlify или просто `index.html`).

## Возможности

- **Обучение** — пошаговые уроки от основ до продвинутого уровня (17 уроков)
- **Песочница** — свободная среда для практики любых команд
- **Задания** — проверка навыков задачами разной сложности (20 заданий)
- **Двуязычный интерфейс** — полная поддержка русского и английского с мгновенным переключением
- **Прогресс** — отслеживание прохождения сохраняется в localStorage
- **40+ команд** — реалистичная симуляция команд Linux
- **Масштабируемость** — новые уроки, задания и команды добавляются без изменения кода

## Доступные команды

### Навигация и файлы
| Команда | Описание |
|---------|----------|
| `ls` | Показать содержимое директории (`-a`, `-l`, `-h`) |
| `cd` | Сменить директорию |
| `pwd` | Показать текущую директорию |
| `mkdir` | Создать директорию (`-p` для вложенных) |
| `rmdir` | Удалить пустую директорию |
| `touch` | Создать пустой файл |
| `rm` | Удалить файлы/директории (`-r`, `-f`) |
| `cp` | Копировать файлы (`-r` для директорий) |
| `mv` | Переместить/переименовать файлы |
| `ln` | Создать ссылки (`-s` для символических) |
| `cat` | Показать содержимое файла (`-n` для нумерации строк) |
| `echo` | Вывести текст (`-n`, `-e`) |
| `head` | Показать первые N строк (`-n`) |
| `tail` | Показать последние N строк (`-n`) |
| `wc` | Подсчёт слов/строк/символов (`-l`, `-w`, `-c`) |
| `diff` | Сравнить два файла |

### Поиск и обработка текста
| Команда | Описание |
|---------|----------|
| `grep` | Поиск по шаблону (`-i`, `-n`, `-r`, `-c`, `-v`) |
| `find` | Поиск файлов (`-name`, `-type`) |
| `sed` | Потоковый редактор (`s/шаблон/замена/флаги`) |
| `awk` | Обработка текста (`{print $N}`) |
| `sort` | Сортировка строк (`-r`, `-n`, `-u`) |
| `uniq` | Удаление дубликатов (`-c`, `-d`) |
| `cut` | Извлечение столбцов (`-d`, `-f`) |
| `tr` | Преобразование/удаление символов |
| `tee` | Чтение stdin, запись в stdout и файлы (`-a`) |

### Система
| Команда | Описание |
|---------|----------|
| `ps` | Показать процессы (`aux`, `-e`) |
| `kill` | Завершить процесс по PID |
| `top` | Монитор системы (снимок) |
| `whoami` | Текущий пользователь |
| `hostname` | Показать имя хоста |
| `date` | Показать дату/время |
| `uname` | Информация о системе (`-a`, `-r`) |
| `clear` | Очистить терминал |
| `env` | Показать переменные окружения |
| `which` | Найти путь к команде |

### Прочее
| Команда | Описание |
|---------|----------|
| `chmod` | Изменить права доступа (`-R`, восьмеричный режим) |
| `sudo` | Выполнить от имени суперпользователя |
| `apt` | Менеджер пакетов (install, remove, update, list, search) |
| `curl` | Получение данных по URL (мок) |
| `tar` | Работа с архивами (мок) |
| `basename` | Извлечь имя файла из пути |
| `dirname` | Извлечь путь без имени файла |
| `seq` | Генерация числовых последовательностей |
| `man` | Справочные страницы |
| `help` | Список доступных команд |
| `history` | История команд |
| `export` | Установить переменную окружения |

### Возможности оболочки (shell)
- **Конвейеры (пайпы)**: `команда1 | команда2 | команда3`
- **Перенаправление**: `>` (запись), `>>` (дозапись)
- **Переменные окружения**: `$HOME`, `$USER`, `export VAR=value`
- **Автодополнение Tab**: команды + пути к файлам
- **История команд**: стрелки вверх/вниз
- **Горячие клавиши**: `Ctrl+C` (отмена), `Ctrl+L` (очистить), `Ctrl+D` (выход)

## Уроки (17)

### Начинающий (5)
1. Первые шаги: навигация (`pwd`, `ls`, `cd`)
2. Работа с файлами (`touch`, `cat`, `echo`, `rm`)
3. Работа с директориями (`mkdir`, `rmdir`, `cp`, `mv`)
4. Чтение файлов (`head`, `tail`, `wc`)
5. Справка по командам (`man`, `--help`)

### Средний (5)
6. Поиск: grep и find
7. Пайпы и перенаправления
8. Обработка текста: sed, awk, sort
9. Права доступа (`chmod`)
10. Системная информация

### Продвинутый (7)
11. Управление пакетами (`apt`)
12. Управление процессами (`ps`, `kill`, `top`)
13. Архивы и сеть (`tar`, `curl`)
14. Символические ссылки (`ln`)
15. Продвинутая обработка текста (`cut`, `tr`, `tee`)
16. Переменные окружения (`env`, `export`, `which`)
17. Составные команды (`seq`, сложные конвейеры)

## Задания (20)

### Лёгкие (5)
Создать файл, навигация по директориям, отсортированный список, чтение файла, создание дерева директорий

### Средние (7)
Найти файлы, подсчитать строки, перенаправление вывода, поиск по шаблону, изменить права, извлечь столбцы, преобразовать текст

### Сложные (8)
Конвейер данных, найти и заменить, управление процессами, аудит системы, комплексный поиск, цепочка трансформаций, анализ данных, обработка логов

## Как запустить

### Вариант 1: Открыть напрямую
Просто откройте `index.html` в браузере. Примечание: некоторые браузеры блокируют `fetch()` с `file://`, поэтому лучше использовать локальный сервер:

### Вариант 2: Локальный сервер
```bash
# Любой из вариантов:
npx serve .
python3 -m http.server 3000
php -S localhost:3000
```
Затем откройте http://localhost:3000

## Деплой на GitHub Pages

### Шаг 1: Создать репозиторий
```bash
cd ubuntu_console_simulator
git init
git add -A
git commit -m "Initial commit: Ubuntu Terminal Trainer"
```

### Шаг 2: Запушить на GitHub
```bash
# Создайте репозиторий на github.com, затем:
git remote add origin https://github.com/YOUR_USERNAME/ubuntu-terminal-trainer.git
git branch -M main
git push -u origin main
```

### Шаг 3: Включить GitHub Pages
1. Перейдите в свой репозиторий на GitHub
2. **Settings** > **Pages** (в левом меню)
3. В разделе **Source** выберите **Deploy from a branch**
4. Выберите ветку **main** и папку **/ (root)**
5. Нажмите **Save**
6. Подождите 1–2 минуты, ваш сайт будет доступен по адресу:
   `https://YOUR_USERNAME.github.io/ubuntu-terminal-trainer/`

### Альтернатива: GitHub CLI
```bash
gh repo create ubuntu-terminal-trainer --public --source=. --push
gh api repos/{owner}/{repo}/pages -X POST -f source.branch=main -f source.path=/
```

## Как расширять

### Добавление нового урока

1. Создайте JSON-файл в `data/lessons/{уровень}/`, например `data/lessons/beginner/06-new-topic.json`:

```json
{
    "id": "basics-new-topic",
    "order": 6,
    "level": "beginner",
    "title": {
        "ru": "Новая тема",
        "en": "New Topic"
    },
    "steps": [
        {
            "instruction": {
                "ru": "Описание шага. Используйте **cmd** для выделения.",
                "en": "Step description. Use **cmd** for emphasis."
            },
            "validate": "^cmd\\s+arg",
            "hint": {
                "ru": "Напишите: cmd arg",
                "en": "Type: cmd arg"
            }
        }
    ]
}
```

2. Пересоберите манифест:
```bash
node tools/build-manifest.js
```

3. Готово! Урок появится автоматически.

**Поля:**
- `id` — уникальный идентификатор (строка)
- `order` — порядок сортировки внутри уровня (число)
- `level` — `"beginner"`, `"intermediate"` или `"advanced"`
- `title` — двуязычный объект `{ ru, en }`
- `steps[]` — массив шагов, каждый содержит:
  - `instruction` — двуязычный текст (поддерживает `**жирный**` и `` `код` `` markdown)
  - `validate` — регулярное выражение для проверки введённой команды
  - `hint` — двуязычная подсказка

### Добавление нового задания

1. Создайте JSON-файл в `data/challenges/{сложность}/`, например `data/challenges/easy/06-new-task.json`:

```json
{
    "id": "easy-new-task",
    "order": 6,
    "difficulty": "easy",
    "title": {
        "ru": "Новое задание",
        "en": "New Task"
    },
    "description": {
        "ru": "Описание задания на русском.",
        "en": "Task description in English."
    },
    "check": {
        "type": "file_exists",
        "path": "/home/user/newfile.txt"
    }
}
```

2. Пересоберите манифест:
```bash
node tools/build-manifest.js
```

**Типы проверок:**
| Тип | Поля | Описание |
|-----|------|----------|
| `file_exists` | `path` | Файл существует по указанному пути |
| `file_content` | `path`, `contains` | Файл содержит указанную подстроку |
| `dir_exists` | `path` | Директория существует по указанному пути |
| `cwd` | `path` | Текущая директория совпадает с указанной |
| `command` | `pattern` | Последняя команда соответствует регулярному выражению |
| `file_permissions` | `path`, `permissions` | Файл имеет указанные права доступа |

### Добавление новой команды

Отредактируйте `js/commands.js`. Внутри `createCommands(fs)` добавьте:

```javascript
// === mycommand ===
commands.mycommand = (args, ctx) => {
    // args — массив аргументов (флаги парсятся вручную)
    // ctx.stdin — входные данные из пайпа (если команда после |)
    // ctx.terminal — экземпляр терминала (для clear и т.д.)
    // ctx.env — переменные окружения
    // ctx.history — массив истории команд
    // fs — виртуальная файловая система (resolve, readFile, writeFile и т.д.)

    // Верните строку для вывода, '' для пустого вывода,
    // или { error: 'сообщение' } для ошибок
    return 'Hello from my command!';
};
```

Затем добавьте команду в вывод `commands.help` и, при желании, в объект `manPages` внутри `commands.man`.

## Технологии

- **Терминал**: [xterm.js](https://xtermjs.org/) v5.5.0 (через CDN)
- **Фронтенд**: Vanilla JavaScript (ES-модули)
- **Стили**: CSS с тёмной темой Ubuntu
- **Прогресс**: localStorage
- **i18n**: встроенные переводы + двуязычные JSON-файлы данных
- **Без npm, без сборщика, без бэкенда** — только статические файлы
